## Sim2Real Gap 이란 ?


## 핵심 질문

- 왜 시뮬레이션에서 완벽하게 작동하는 로봇이 현실에서는 실패하는가 ?
- 시각, 물리, 시스템 레벨에서 발생하는 격차의 본질은 무엇인가?
- 이 격차를 극복하기 위한 3대 방법론(Sys-ID, DR, DA)의 원리는?
- 파운데이션 모델과 미분 가능 시뮬레이터가 가져올 패러다임 변화는 ?

## 오늘의 학습 목표

- Sim2Real Gap의 3가지 원인 (시각, 물리, 시스템)을 구분하고 각각의 영향 분석
- 3대 해결 방법론 (System ID, Domain Randomization, Domain Adaptation)의 원리와 적용 조건 이해
- 문제-기술 적합성에 따른 최적 솔루션 선택은 ?


### Sim2Real Gap 정의

시뮬레이션에서 학습한 모델/정책이 실제 로봇으로 이전될 때 발생하는 성능 저하 현상

### 영향

단순 성능 저하부터 시스템의 완전한 오작동('치명적 실패')까지 다양함

### 근본 원인

학습 데이터(시뮬레이션)와 테스트 데이터(현실)의 분포가 다른 '영역 이동(Domain Shift)' 현상

### 왜 중요한가?

로보틱스 기술의 상용화, 안전성, 경제성 확보의 핵심 과제 

## 시각적 불일치 (Visual Mismatch)

### 렌더링의 한계

- 물리적으로 정확한 조명(그림자, 반사) 시뮬레이션의 계산 비용 문제
- 물체의 재질, 질감 표현의 부정확성
- 시뮬레이션 특유의 시각적 결함(artifact)에 모델이 과적합될 위험

### 콘텐츠 갭 (Content Gap)

- 3D 모델 자체의 기하학적 왜곡
- 텍스처 결함 및 부정확한 재질 표현

### 결과

비전 기반 정책이 실제 카메라 영상의 미세한 차이에 오작동 


## 물리적 동역학의 차이 (Physical Divergence)

### 접촉이 풍부한 환경의 어려움 High Impact 

- 물체 조작, 부품 조립 등에서 문제 증폭
- 마찰, 변형, 충돌 등 복잡한 비선형 현상을 정확히 모델링하기 어려움
- 특히 정적 마찰(Static Friction)이 로봇 성능에 미치는 영향 과소평가

### 비모델링 동역학 (Non-modeled Dynamics)

- 시뮬레이션이 예측하지 못하는 실제 물리 현상
- 케이블 장력, 백래시, 유연체 변형 등

**Agilibity Robotics 의 Digit 로봇 사례** 

- 시뮬레이션: 안정적인 보행
- 현실: 발이 미끄러지거나 예상치 못한 충돌
- 원인: 발가락 충격(toe impact)의 혼돈 특성
- 해결: 6개월간 물리 모델 개선 작업

### 작은 물리적 차이가 시간이 지나면서 증폭되어 완전히 다른 결과를 초래

# 시스템 수준의 불일치 (System-Level Discrepancies)

### 센서 노이즈 MEDIUM IMPACT 

- 실제 센서 노이즈 프로파일을 시뮬레이션에서 정확히 복제하기 거의 불가능
- 가우시안 노이즈 모델 vs 실제의 복잡한 노이즈 패턴
- 센서 드리프트, 온도 영향, 전자기 간섭등

### 액추에이터 지연 시간 (Latency) HIGH IMPACT 

- 시뮬레이션 : 인식 -> 결정 -> 행동이 즉각적 (동기식)
- 현실 : 감지, 계산, 통신, 구동 모든 단계에서 지연 (비동기식)
- 영향 : 100ms 지연 = 과거 정보 기반 행동

시간 지연의 누적 효과 (총 지연 ~100ms)

=> 로봇은 항상 '과거의 정보'에 기반하여 행동

- 100ms 지연 = 시속 10km/h로 움직이는 로봇이 28cm 이동
- 고속 작업에서는 치명적인 오차 발생 가능

# Sim2Real Gap을 극복하기 위한 세가지 핵심 접근법

### 01. 시스템 식별

시뮬레이터를 현실과 똑같이 만들자

System Identification

- 실제 데이터로 시뮬레이터 파라미터 최적화
- 고충실도 디지털 트윈 구축
- Real2Sim 접근법

### 02. 영역 무작위화

현실의 불확실성에 강인한 정책을 만들자

Domain Randomization

- 파라미터를 무작위로 변경
- 다양한 환경에서 학습
- 일반화 능력 향상

### 03. 영역 적응

시뮬레이션 데이터를 현실처럼 보이게 만들자

Domain Adaptation

- GAN 등으로 데이터 변환
- 특징 수준 정렬
- 스타일 전이 기법


### 방법 1 - 시스템 식별 (System Identification)

실제 시스템의 '디지털 트윈'에 가까운 고충실도 시뮬레이터 구축

원리 (Real2Sim)

- 실제 로봇의 궤적 데이터를 수집
- 시뮬레이션과 실제 데이터 간 오차 간소화
- 시뮬레이터 파라미터 자동 조정

장점

- 원칙에 입각한 접근
- 해석 가능한 결과
- 고정밀 정책 학습 가능
- 특정 로봇에 최적화

단점

- 높은 비용과 시간
- 전문가 지식 필요
- 복잡한 시스템 어려움
- 새 환경마다 재수행

최신연구동향

- COMPASS (2023) : 인과관계 발견으로 자동화
- 미분 가능 시뮬레이터 : 역전파로 효율적 최적화
- 64개 파라미터까지 확장 가능

### 방법 2 - 영역 무작위화 (Domain Randomization)

#### **완벽한 시뮬레이터 대신, 무작위화된 수많은 환경에서 강인한 정책 학습**

- 매 학습 에피소드마다 환경 파라미터를 무작위로 변경
- 정책이 특정 시뮬레이션 결함에 과적합되는 것을 방지
- "현실은 무작위화 분포 내의 한 샘플"이라는 가정

장점

- Zero-shot 전이 가능
- 실제 데이터 불필요
- 모델링 어려운 문제에 효과적
- 구현이 상대적으로 간단

단점

- 차선책/보수적 정책 가능
- 무작위화 범위 튜닝 필요
- 과도한 무작위화는 학습 방해
- 계산 비용 증가

OpenAI 루빅스 큐브 성공 사례

- ADR (Automatic DR) 사용
- 정책 성능에 따라 무작위화 범위 자동 확장
- 시뮬레이션에서만 학습, 실제 로봇 손으로 직접 전이
- 결과: 60% 성공률 달성

### 방법 3 - 영역 적응 (Domain Adaptation)

#### **시뮬레이션과 현실 데이터의 통계적 분포를 직접 정렬**

#### **픽셀 수준 적응**

#### **CycleGAN, RCAN 등**

* Sim -> Real 이미지 변환
* 포토리얼리스틱 렌더링
* 스타일 전이 기법

#### **특징 수준 적응**

#### **적대적 학습**

* 도메인 불변 특징 학습
* 공유 표현 공간 구축
* 분류기 혼동 최대화

#### **주요 변호나 특징**

**재질 & 텍스처**

**단색 -> 그라디언트, 금속 질감**

**조명 & 그림자**

**균일한 빛 -> 자연스러운 명암, 그림자**

**반사 & 하이라이트**

**매트한 표면 -> 반사광, 광택 효과**

**카메라 특성**

**완벽한 선명도 -> 실제 렌즈 효과 **

#### **3가지 방법론 비교 분석**

#### **문제-기술 적합성 (Problem-Technique Fit)**

**이 방법론들은 상호 배타적이지 않으며, 최근 연구는 이들을 결합하는 하이브리드 접근법으로 발전 중**

**성공의 열쇠는 문제의 특성에 맞는 기술 조합을 선택하는 것!**

#### 정교한 조작

접촉이 많고 정밀도가 중요한 작업
예: 조립, 삽입, 정밀 그리핑
최적 DR + DA**

#### 산업용 자동화

반복적이고  구조화된 환경
예: 픽앤플레이스, 팔레타이징
최적 Sys-ID + DR

#### 자율주행

복잡한 시각 환경, 다양한 날씨
예: 도심 주행, 주차
최적 DA + DR**

### **Sim2Real 실제 적용 사례**

#### **[사례 1] OpenAI의 루빅스 큐브**

#### **문제**

**수 많은 동시적, 복잡한 접촉으로 인해 물리**

**모델링이 거의 불가능한 '정교한 조작' 과제**

#### **해결 - Automatic Domain Randomization (ADR)**

* 정확한 물리 모델링을 우회하는 전략적 선택
* 정책 성능에 따라 무작위화 범위 자동 확장
* 커리큘럼 학습 방식

#### **결과**

* '창발적 메타 학습' 능력 획득
* 실제 환경 동역학에 빠른 적응
* Zero-shot Sim2Real 이전 성공

### **정리**

* Sim2Real Gap은 **시각, 물리, 시스템**의 복합적인 문제
* **Sys-ID, DR, DA** 는 상호 보완적 전략으로 문제 특성에 맞게 선택
* 성공 사례들은  **'문제-기술 적합성'** 의 중요성을 입증

#### **실무 적용 가이드**

* **정교한 조작 작업**
  * DR + DA 조합으로 시각적 충실도와 강인성 확보
* **산업용 자동화**
  * Sys-ID + DR로 디지털 트윈 구축
* **자율주행 / 드론**
  * DA + 대규모 DR로 다양한 환경 대응

#### **3가지 원칙**

* 측정 가능한 것만 개선 가능 : 정량적 격차 분석부터 시작
* 문제-기술 적합성이 핵심 : 만능 해법은 없다
* 반복적 개선이 답 : Sim <-> Real 지속적 피드백 루프

**완벽한 시뮬레이션을 추구하기보다는, 불완전한 시뮬레이션에서도 강인하게 작동하는**
**시스템을 만드는 것이 진정한 해답**
